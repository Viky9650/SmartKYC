📄 SmartKYC — Document Upload Pipeline

From file upload to extracted fields in the database

Step 1 · Entry Point
POST /api/documents/upload/{case_id}
Frontend sends file + optional document_type hint · Accepted: JPEG, PNG, WebP, TIFF, GIF, PDF
Step 2 · Validate & Save
Validate case exists · Save raw bytes to disk
File stored as {uuid}{ext} · UUID becomes document_id
Step 3 · Document Type Detection
Classify document schema
Use frontend hint if provided · Otherwise parse filename for keywords
e.g. "ru_passport_viktor.pdf" → RU_PASSPORT · "aadhaar_card.jpg" → IN_AADHAAR
20+ schemas: IN_PAN · IN_AADHAAR · RU_PASSPORT · GB_PASSPORT · AE_EMIRATES_ID · CN_ID_CARD…
Step 4 · File Type Check
Is it a PDF?
✓ YES — PDF
PDF Pre-processing
pdfminer text extract + pdf2image render
pdfminer → fast text from searchable PDFs
pdf2image → render page 1 at 200 DPI → PNG
Scanned PDFs: no text, image only
✗ NO — Image
Image Pre-processing
Use raw image bytes directly
JPEG / PNG / WebP / TIFF passed straight to extraction
Step 5 · Primary Path — Gemini Vision
🔮 Google Gemini Vision API
Image + schema-aware prompt → structured JSON
Returns: fields · per-field confidence · MRZ lines · document_quality
Truncated JSON → partial recovery parser (regex salvage)
Step 6 · Vision Result Check
Did Vision return fields?
(fails if GEMINI_API_KEY not set, network error, or parse failure)
✓ Fields extracted
Vision + MRZ Merge
Run MRZ parser on Vision's MRZ lines
Merge MRZ fields into Vision fields at conf 0.97
MRZ wins only for fields Vision didn't extract
✗ Vision unavailable
Step 7 · OCR Fallback — Tesseract
🔎 Tesseract OCR
Greyscale + sharpen · 3× PSM modes (3, 6, 4)
+ dedicated MRZ pass (A-Z 0-9 < whitelist)
All passes concatenated → raw text blob
Step 8 · MRZ Parser (ICAO TD3)
🛂 Machine-Readable Zone decode
Detects 44-char P<XXX line pairs
Decodes: surname · given names · DOB · nationality · expiry · doc number · sex
Step 9 · Regex Parsers
Document-specific field extraction
IN_AADHAAR → 12-digit Aadhaar · DOB · gender · address
IN_PAN → AAAAA9999A pattern · father name
COMPANY_REG → reg number · directors · address
Generic → passport number · dates · name · nationality
Step 10 · Result Assembly
Sanitise · compute full_name · score confidence
OCR noise stripped from all field values
full_name assembled: full_name → name → given_names + surname
overall_confidence = average of all per-field scores
Extraction SourceConfidence
MRZ Parser 0.97
Gemini Vision 0.96
pdfminer (text PDF) 0.90
Tesseract OCR 0.82
Regex parsers 0.80
Step 11 · Database Persistence
Write Document record + DocumentExtraction rows
Document → full extracted_data JSON blob (fields, confidences, schema, method, quality)
DocumentExtraction → one row per field with individual confidence score
Step 12 · API Response
✅ Return to frontend immediately
document_id · document_type · filename · file_size
extracted_data (persisted) · extraction (full live result)
Frontend renders extraction panel — no second round trip needed
